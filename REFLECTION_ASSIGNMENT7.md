# Reflection
## Assignment 7: GitHub Project Templates and Kanban Board

---

## Introduction
This assignment provided practical experience in using GitHub Projects to implement a Kanban board for Agile workflow management. It involved selecting a suitable template, customizing it to fit system requirements, and organizing tasks based on user stories from previous assignments.

---

## Challenges in Template Selection
One of the main challenges was selecting the most appropriate GitHub project template. GitHub provides several templates, such as Basic Kanban, Automated Kanban, and Bug Triage, each designed for different purposes.

Initially, it was difficult to determine which template best supported Agile development. The differences between templates were not always obvious, especially with GitHub’s updated interface where the “Automated Kanban” template is now integrated into the standard “Kanban” template. Understanding how automation features worked required additional exploration.

Choosing the Kanban template required evaluating how well it supports workflow tracking, task movement, and integration with GitHub Issues created in previous assignments.

---

## Challenges in Customizing the Kanban Board
Another challenge was customizing the Kanban board to align with real-world Agile workflows. While the default columns (Backlog, To Do, In Progress) were useful, additional columns such as **Testing** and **Blocked** had to be added manually.

Deciding which columns to include required understanding the software development lifecycle. For example:
- The **Testing** column ensures that completed tasks are verified before being marked as done.
- The **Blocked** column helps identify tasks that cannot proceed due to dependencies or issues.

Organizing tasks across columns also required careful planning to ensure that the workflow accurately reflected task progress and did not overload any single stage.

---

## Comparison with Other Tools (Trello and Jira)
GitHub Projects provides a simple and integrated approach to Agile project management. Compared to other tools:

- **Trello**  
  Trello offers a highly visual and user-friendly Kanban board with drag-and-drop functionality. It is easy to use for beginners but lacks deep integration with code repositories compared to GitHub.

- **Jira**  
  Jira is a more advanced Agile tool with features such as sprint management, reporting, and detailed backlog tracking. However, it can be complex and requires more setup compared to GitHub Projects.

- **GitHub Projects**  
  GitHub Projects integrates directly with repositories, issues, and code, making it highly suitable for developers. While it is less feature-rich than Jira, it is more efficient for smaller projects and individual development work.

---

## Lessons Learned
This assignment improved my understanding of how Kanban boards are used to visualize workflow and manage tasks in Agile environments. It also highlighted the importance of selecting the right tools and customizing them to suit project requirements.

Additionally, the integration between GitHub Issues and the Kanban board demonstrated how user stories can be effectively tracked throughout the development process.

---

## Conclusion
Overall, this assignment provided valuable experience in applying Agile principles using GitHub Projects. It emphasized the importance of workflow visualization, task organization, and tool selection in software development. Despite some challenges, the customized Kanban board successfully supports the development process of the Student Early Warning System.
