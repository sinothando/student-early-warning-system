# User Stories
## Student Early Warning System

---

| Story ID | User Story | Acceptance Criteria | Priority |
|---------|-----------|--------------------|---------|
| US-001 | As a student, I want to view my academic performance so that I can monitor my progress | Performance data is displayed clearly | High |
| US-002 | As a student, I want to receive alerts so that I know when I am at risk | Alerts are generated when thresholds are met | High |
| US-003 | As a lecturer, I want to record attendance so that student participation is tracked | Attendance is saved successfully | High |
| US-004 | As a lecturer, I want to upload grades so that student performance is updated | Grades are stored correctly | High |
| US-005 | As a lecturer, I want to view a dashboard so that I can monitor student performance | Dashboard loads within 2 seconds | High |
| US-006 | As an advisor, I want to monitor flagged students so that I can provide support | At-risk students are displayed | High |
| US-007 | As an advisor, I want to provide intervention so that students can improve | Intervention is recorded | Medium |
| US-008 | As an admin, I want to manage users so that system access is controlled | Users can be added/removed | Medium |
| US-009 | As the system, I want to detect risk automatically so that students are flagged early | Students below thresholds are flagged | High |
| US-010 | As the system, I want to generate alerts so that users are notified | Alerts are triggered correctly | High |
| US-011 | As IT staff, I want system uptime so that users can access the system reliably | System uptime ≥ 99% | Medium |
| US-012 | As management, I want reports so that I can analyze student performance | Reports generated successfully | Low |
