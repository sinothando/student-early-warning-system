# Reflection

One of the major challenges in this assignment was translating functional requirements into clear use cases. While the requirements defined what the system should do, converting them into user interactions required deeper understanding of system behavior.

Another challenge was ensuring alignment between stakeholders, functional requirements, and use cases. Each use case had to directly support stakeholder needs identified in Assignment 4, which required careful mapping.

Designing test cases was also challenging because each test needed to validate a specific requirement. It was important to ensure that test cases were clear, measurable, and aligned with expected system behavior.

Additionally, balancing simplicity and completeness was difficult. The system needed to remain feasible for a single developer while still demonstrating all required software engineering concepts.

Overall, this assignment improved my understanding of how requirements are transformed into system interactions and validated through testing. It highlighted the importance of traceability between stakeholders, requirements, and system design.
