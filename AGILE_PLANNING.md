## 3. User Stories Summary

User stories were created based on functional requirements and use cases to represent user needs in Agile format.

Example:
"As a student, I want to view my academic performance so that I can monitor my progress."

These stories follow the INVEST principles and are prioritized based on stakeholder importance.

---

## 4. Product Backlog Summary

The product backlog organizes user stories using MoSCoW prioritization:

- **Must-have:** Core features required for MVP (e.g., risk detection, attendance tracking)
- **Should-have:** Important but not critical features
- **Could-have:** Optional enhancements
- **Won’t-have:** Deferred features

Effort estimation was done using story points (1–5 scale).
